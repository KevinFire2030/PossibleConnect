# Qdrant Vector Store 노드 Description 작성 가이드

## Description이란?

**Description**은 AI Agent가 이 도구(Tool)를 사용할 때 이해하는 설명입니다. AI Agent가 언제 이 도구를 사용해야 하는지, 어떤 정보를 제공하는지 알 수 있도록 명확하게 작성해야 합니다.

## Description 작성 원칙

### 1. 명확성
- 도구의 목적을 명확하게 설명
- 어떤 정보를 제공하는지 명시

### 2. 맥락 제공
- 언제 이 도구를 사용해야 하는지 설명
- 어떤 종류의 질문에 유용한지 명시

### 3. 간결성
- 너무 길지 않게 작성
- 핵심 정보만 포함

## RAG 시스템용 Description 예시

### 예시 1: 기본 설명 (권장)

```
동아리 문서(소개, 회칙, 사업계획, FAQ)에서 관련 정보를 검색합니다. 
사용자의 질문과 관련된 문서 내용을 찾아 제공합니다.
```

### 예시 2: 상세 설명

```
동아리 관련 문서 데이터베이스에서 의미 기반 검색을 수행합니다. 
동아리 소개, 회칙, 2026년 사업계획, FAQ 등의 문서에서 
사용자 질문과 관련된 내용을 찾아 반환합니다.
```

### 예시 3: 간결한 설명

```
동아리 문서에서 관련 정보를 검색하여 제공합니다.
```

### 예시 4: 영어 버전

```
Search and retrieve relevant information from club documents 
including introduction, bylaws, business plans, and FAQ.
```

## Description 작성 팁

### 포함해야 할 내용

1. **데이터 소스**
   - 어떤 문서/데이터를 검색하는지
   - 예: "동아리 문서", "회칙", "FAQ"

2. **검색 방법**
   - 의미 기반 검색인지, 키워드 검색인지
   - 예: "의미 기반 검색", "semantic search"

3. **사용 시기**
   - 언제 이 도구를 사용해야 하는지
   - 예: "사용자의 질문과 관련된 정보가 필요할 때"

### 피해야 할 내용

- ❌ 너무 기술적인 용어
- ❌ 도구의 내부 동작 방식
- ❌ 불필요한 세부사항

## AI Agent가 Description을 사용하는 방법

### 1. 도구 선택
- 사용자 질문을 분석
- Description을 읽고 적절한 도구 선택
- "동아리 관련 질문" → Qdrant Vector Store 도구 사용

### 2. 검색 쿼리 생성
- 사용자 질문을 검색 쿼리로 변환
- 의미 기반 검색 수행

### 3. 결과 활용
- 검색된 문서를 컨텍스트로 사용
- 사용자 질문에 답변 생성

## 실제 사용 예시

### 사용자 질문
```
"동아리 가입 방법은?"
```

### AI Agent의 판단
1. Description 읽기: "동아리 문서에서 관련 정보를 검색"
2. 도구 선택: Qdrant Vector Store 사용
3. 검색 수행: "동아리 가입 방법" 관련 문서 검색
4. 결과 활용: 검색된 문서를 바탕으로 답변 생성

## Description 수정 방법

### n8n에서 수정

1. **Qdrant Vector Store** 노드 선택
2. **Parameters** 탭
3. **Description** 필드 찾기
4. 원하는 설명 입력
5. **Save** 클릭
